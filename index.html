今天先把工程框架搭建起来再记录一个点亮led的工程 顺便复习一下cubemx的使用 话不多说 现在开始

09-07 led点亮

首先打开cubemx软件

点击File下拉菜单下的New Project

在搜索栏输入427ii后选择STM32F427IIHx

选择System Core中的RCC(Double-click) 选择HSE中的Crystal/Ceramic Resonator(晶体/陶瓷谐振器)

接下来配置时钟树 选择Clock Configuration

如图进行相关配置

查看Robomaster开发板A型原理图后得知

开发板的8个led分别连接在PG1-PG8引脚上

回到Pinout&Configuration中继续配置gpio


搜索栏直接输入引脚搜索

点击对应引脚配置其为GPIO_Output模式

同理配置另外7个引脚

配置完成后如图

配置完成后 我们查看一下gpio配置 这里我们暂不更改 直接使用默认配置即可

一切配置完成后 我们生成代码


自定义名称、路径、和编译工具

这里选择为每个部分生成独立的.c与.h文件 方便以后的管理

最后点击GENERATE CODE生成代码

等待完成打开工程

编译运行后 0 error 0 waring

将程序烧录到开发板


按下开发板reset键后 led灯成功点亮

09-08 串口配置

继续打开上次生成的工程


选择USART6配置为Asynchronous(异步通信模式)


默认波特率为115200

配置完成后生成代码 打开工程 编译


同样没有报错 这时我们发现工程目录中增加了usart.c等相关文件

接下来我们根据HAL库提供的函数实现串口发送和接收的功能以方便日后的调试


在查询相关资料后 找到一个串口发送的函数 原型如图
我们在主函数中调用此函数

由于使用到了strlen函数 我们需要引入string.h头文件

编译成功后烧录程序 打开串口调试工具显示如图


注意串口选择部分要对应cubemx中的设置

串口接收函数原型如图


据此我们开发一个电脑发送数据给主控板 主控板再发回给计算机的程序

首先我们先定义一个接收串口数据的数组(暂且设其长度为20)并将其初始化为0

之后我们在主循环中通过HAL库串口接收函数接收数据再发送出去

编译后得到一个waring


原来是参数类型不匹配 我们将其进行类型转换

waring消失

打开串口助手进行调试


发现单片机一直发送数据 这显然不是我们所希望的 因此 我们在原有的基础上进行修改 在发送一次数据后将数组清空
